Declaración de Accesibilidad

A) Nivel de Conformidad Alcanzado

WCAG 2.2 Nivel A

Este sitio web cumple con los niveles de accesibilidad AA de las WCAG 2.2, asegurando que el contenido sea accesible para personas con diferentes tipos de discapacidad.

  • ✅ Navegación por teclado completa
  • ✅ Contraste de colores adecuado
  • ✅ Textos alternativos en imágenes
  • ✅ Estructura semántica correcta
  • ✅ Formularios accesibles

B) Accesibilidad en la Navegación

1. Navegación por Teclado

Implementación de navegación completa mediante teclado con indicadores visuales de focus.

/* Estilos para focus visible */
a:focus, button:focus, input:focus {
    outline: 2px solid #2272FF;
    outline-offset: 2px;
}

nav ul li a:focus {
    outline: 2px solid coral;
    background-color: rgba(255, 127, 80, 0.1);
}

2. Atajos de Teclado (Accesskeys)

Implementación de accesskeys para acceso rápido a secciones principales del sitio.

<nav>
  <ul>
    <li><a href="index.html" accesskey="I">Inicio</a></li>
    <li><a href="accesibilidad.html" accesskey="A">Accesibilidad</a></li>
  </ul>
</nav>

3. Estructura Semántica Clara

Uso de elementos HTML5 semánticos para una estructura comprensible mediante lectores de pantalla.

<header>
  <nav>
    <main>
      <section>
      <article>
    <aside>
    <footer>

D) Criterios de Conformidad

1. Contraste de Color (WCAG 1.4.3)

Criterio de Conformidad: 1.4.3 Contraste (Mínimo) - Nivel AA

Técnica: G18 - Asegurar que existe un contraste mínimo entre el texto y el fondo

Implementación: Se han seleccionado combinaciones de colores con ratio de contraste superior a 4.5:1

Texto ejemplo - Contraste 4.8:1

2. Navegación por Teclado (WCAG 2.1.1)

Criterio de Conformidad: 2.1.1 Teclado - Nivel A

Técnica: G202 - Asegurar el control del teclado para todas las funcionalidades

Implementación: Todos los elementos interactivos son accesibles mediante teclado con indicadores visuales de focus

.cartaproducto:focus,
.imagen-contenedor:focus {
    outline: 2px solid #2272FF;
    outline-offset: 2px;
}

E) Principios de Jakob Nielsen

1. Visibilidad del estado del sistema

¿Cómo se aplica? El usuario siempre sabe en qué parte del sitio se encuentra mediante:

  • Indicadores de navegación activos
  • Títulos de página claros
  • Ruta de navegación visual
<nav>
  <ul>
    <li><a href="index.html" class="activo">Inicio</a></li>
    <li><a href="accesibilidad.html">Accesibilidad</a></li>
  </ul>
</nav>

<header>
  <h1>TuttifruttiReviews</h1>
</header>

2. Consistencia y estándares

¿Cómo se aplica? Mantenimiento de patrones consistentes en toda la web:

  • Misma estructura de navegación en todas las páginas
  • Paleta de colores uniforme
  • Comportamientos predecibles en elementos interactivos
/* Variables CSS para consistencia */
:root {
    --color-fondo: #F5F7FF;
    --color-fondonavegacion: rgba(1, 37, 125);
    --color-footer: #2272FF;
}

/* Estilos consistentes para botones */
.btn {
    padding: 12px 30px;
    border: none;
    border-radius: 6px;
    font-family: "Roboto", sans-serif;
}

3. Prevención de errores

¿Cómo se aplica? Diseño que previene errores del usuario:

  • Validación de formularios en tiempo real
  • Confirmaciones para acciones destructivas
  • Mensajes de error claros y específicos
<input type="tel" id="telefono" name="telefono" required
       pattern="[0-9]{9}" placeholder="Ej: 612345678"
       title="9 dígitos numéricos">
<span class="texto-ayuda">9 dígitos numéricos</span>

<textarea id="observaciones" name="observaciones" 
          maxlength="500"
          placeholder="Máximo 500 caracteres"></textarea>
<span class="texto-ayuda">Máximo 500 caracteres</span>
